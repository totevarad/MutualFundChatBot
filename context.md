# Project Context: Mutual Fund FAQ Assistant (Facts-Only Q&A)

## Overview
The goal of this project is to build a **facts-only FAQ assistant** for mutual fund schemes, using **Groww** as the reference product context. The assistant is designed to answer objective, verifiable queries related to mutual funds by retrieving information exclusively from official public sources, such as AMC (Asset Management Company) websites, AMFI, and SEBI.

The system strictly avoids providing investment advice, opinions, or recommendations. Every response must include a single, clear source link and adhere to defined constraints around clarity, accuracy, and compliance.

---

## Core Objective
Design and implement a lightweight **Retrieval-Augmented Generation (RAG)**-based assistant that:
- Answers factual queries about mutual fund schemes.
- Uses a curated corpus of official documents.
- Provides concise, source-backed responses.

---

## Target Users
1. **Retail Investors**: Individuals looking to compare mutual fund schemes based on objective parameters.
2. **Customer Support & Content Teams**: Internal teams handling repetitive, factual mutual fund queries.

---

## Scope of Work

### 1. Corpus Definition
To construct the retrieval database, the system must focus on:
- **One Selected Asset Management Company (AMC)** (e.g., SBI Mutual Fund, HDFC Mutual Fund, ICICI Prudential Mutual Fund).
- **3–5 Mutual Fund Schemes** ensuring category diversity (e.g., Large-Cap, Flexi-Cap, ELSS, Debt, or Hybrid).
- **15–25 Official Public URLs**, which must include:
  - Scheme Factsheets
  - Key Information Memorandums (KIM)
  - Scheme Information Documents (SID)
  - AMC FAQ/help pages
  - AMFI/SEBI guidance pages
  - Statement and tax document download guides

### 2. FAQ Assistant Requirements
The assistant must support facts-only queries including, but not limited to:
- Expense ratio of a scheme
- Exit load details
- Minimum SIP amount
- ELSS lock-in period
- Riskometer classification
- Benchmark index
- Process to download statements or capital gains reports

### 3. Response Format & Style
Every factual response generated by the assistant **must** adhere to these strict visual and textual rules:
- **Length Constraint**: Limited to a maximum of **3 sentences**.
- **Citation**: Include **exactly one** valid citation link to the official source document.
- **Footer**: Include a standard footer in the exact format:
  > *Last updated from sources: <date>*
- **No-Information Fallback**: If the system cannot find the answer in the retrieved context:
  - State that you do not have the information in your current sources.
  - Dynamically search `data/sources.json` for keywords in the user query and suggest the most relevant official URL as a clickable link fallback (e.g. routing to HDFC KYC, account statements, FATCA forms, or contact details depending on user intent).

### 4. Refusal Handling
The assistant **must** decline to answer non-factual, subjective, or advisory queries, such as:
- *"Should I invest in this fund?"*
- *"Which fund is better?"*
- *"Will this fund give 15% returns next year?"*

**Refusal Response Requirements:**
- Must be polite and clearly worded.
- Must reinforce the facts-only limitation of the assistant.
- Must provide a relevant educational link (e.g., AMFI or SEBI investor education resources).

## Strict Constraints & Security Rules

### 🚫 Data Privacy & Security (Zero-Collection Policy)
To ensure compliance and security, the system **must not** collect, store, or process:
- PAN (Permanent Account Number) or Aadhaar numbers
- Bank account numbers
- OTPs (One-Time Passwords)
- Email addresses or phone numbers

### 🚫 Content Restrictions
- **No Investment Advice**: Under no circumstances should the system make recommendations or express opinions.
- **No Return Calculations**: The system must not compute or project expected performance or perform direct scheme-to-scheme return comparisons.
- **Performance Queries**: For any performance-related queries, the assistant must only provide a direct link to the official scheme factsheet.

### 🔍 Transparency & Compliance
- All answers must be short, purely factual, and verifiable.
- Every answer must include a working source link and the last updated date.
- Do **not** use third-party blogs, news aggregators, or unofficial sources. Only official AMC, AMFI, and SEBI sources are permitted.

## Technical Implementation Guide (RAG)

### Recommended Stack
- **Frontend**: Single-page application or minimal responsive layout.
- **Backend/RAG**: Python (e.g., FastAPI, LangChain, or Llamaindex) or Node.js.
- **Data Ingestion**: Parsers for PDFs/HTML pages (Factsheets, SID, KIM).
- **Vector Store**: Lightweight vector database (e.g., ChromaDB, FAISS, or in-memory vector search) for storing semantic chunks of the official documents.
- **LLM/API Integration**: A reliable model to process retrieved chunks and generate facts-only answers matching the 3-sentence constraint.
